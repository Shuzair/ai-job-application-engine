# Job Description
Job Title: MLOps / Data Engineer — Data & Annotation Pipelines
Location: Remote, Pakistan (only)
Timezone: US Eastern Time
When To Start: Immediately

About the Role
We are looking for a skilled MLOps / Data Engineer to join our company to work for a fast-moving AI product company working on cutting-edge Computer Vision applications. The existing data and annotation infrastructure is largely in place, your primary responsibility will be to maintain, improve, and extend these systems rather than build from scratch. You will work closely with a small, highly technical team and are expected to work independently with minimal supervision.

This is not a research role. You will be hands-on with production data systems every day.

What You Will Be Doing
- Maintaining and improving production data pipelines built in Python
- Working with dataset versioning and reproducibility systems, ensuring datasets are properly versioned, hashed, and released atomically
- Managing and extending annotation platform workflows, working with annotation tools, COCO format data, and format adapters between different annotation systems
- Contributing to the ongoing development of a custom annotation platform, this includes both backend and some frontend work
- Maintaining data quality systems, deduplication, drift detection, data profiling, embeddings based clustering, and sanity checks
- Handling raw data triage and curation, sorting, classifying, and validating inbound data, catching mislabeled, duplicated, or contaminated samples
- Implementing and maintaining PII handling and de-identification pipelines for sensitive data.
- Building and maintaining auditable intake pipelines that ensure data integrity at every step
- Managing cloud infrastructure for data, AWS S3, IAM policies, and related services
- Containerizing and deploying data services using Docker.
- Building and maintaining pipeline orchestration using Airflow, Prefect, or Dagster
- Working with configuration management using Hydra.

What We Are Looking For

Must Have:
- Strong production Python, not just scripts or notebooks, but robust maintainable code that runs reliably in production
- Deep understanding of data and how it gets messy in real world systems, deduplication, validation, quality issues, contamination
- Experience with dataset versioning tools, DVC, LakeFS, or equivalent
- Familiarity with annotation platforms, V7 Darwin, CVAT, Labelbox, or Scale — and COCO format in and out
- Experience with raw data curation, intake pipelines, data triage, PII handling and de-identification
- AWS experience: S3, IAM, and related services
- Docker for containerizing data services.
- Pipeline orchestration, Airflow, Prefect, or Dagster
- Scikit-learn and Pandas for data profiling and analysis

Good to Have:
- Experience with Hydra for configuration management
- Frontend experience, React or similar, for annotation platform work
- Experience with 3D data annotation or 3D annotation platform development
- Medical or healthcare domain data experience.
- Experience with content hashing, manifests, and atomic data releases

What Kind of Person Thrives in This Role
- You are comfortable working independently and do not need to be told what to do next
- You communicate proactively, you share updates without being asked
- You take ownership, if something is broken you fix it without waiting for someone else to notice
- You are comfortable with ambiguity and messy real world data
- You can read existing code and understand it quickly without needing someone to walk you through it.
